# 🗺️ ai-home 路线图（按你当前目标重排）

目标基线（不再含歧义）：
1. 只使用单一全局会话源（`~/.codex` / `~/.claude` / `~/.gemini`），不引入任何中间共享目录。
2. 配置资产支持加密导出/导入，跨机器无缝恢复。
3. 导出/导入只针对账号配置资产，不包含会话历史与运行态缓存。
4. 规划 Rust 优先 GUI，覆盖 Windows / Linux / macOS。
5. 支持远程项目与手机端控制，底层支持在虚拟环境中运行 codex/claude/gemini。

---

## Milestone 1: CLI 稳定基线（当前优先）
**目标**：把“单一全局源 + 纯读命令无副作用”打牢，避免后续 GUI/远程能力建立在不稳定基础上。

**范围**
- 单一会话源：每个工具会话都只落在其原生全局目录。
- 健康检查：新增 `aih doctor`，检查软链、目录权限、配置完整性。
- 默认切换最小化：`set-default` 只影响 `aih` 默认账号，不引入全局劫持动作。

**验收标准**
- `aih <cli> ls` 为纯读操作，不触发全账户写入。
- `aih <cli> set-default <id>` 不破坏原生目录结构。
- `doctor` 能识别目录异常、权限异常与关键配置缺失。

---

## Milestone 2: 加密导出/导入（配置资产）
**目标**：跨机器“账号配置无缝迁移”，并保持边界清晰。

**范围**
- 导出对象：`profiles`（Token/API Key/环境变量/账号配置）。
- 不导出对象：会话历史与项目运行态缓存（避免跨机项目不一致）。
- 加密模式：`AES-256-GCM`（密码）与 `age + SSH key`（公钥）。
- 导入策略：冲突检测、覆盖/跳过、详细报告。

**验收标准**
- A 机器导出 -> B 机器导入后，账号可直接启动并认证通过。
- 导入报告清楚标出：新增、覆盖、跳过、失败项。
- 全流程有非交互脚本模式（便于 CI/自动化运维）。

---

## Milestone 3: Rust 桌面端（跨平台 GUI）
**目标**：推出生产可用的桌面控制台，覆盖 Win/Linux/macOS。

**技术方向**
- Rust + Tauri（后端与系统能力），前端 UI 可选 React。

**核心功能**
- 账号与工具看板：状态、默认账号、使用额度、最近操作。
- 一键启动终端会话（codex/claude/gemini）。
- 配置导出/导入可视化。
- 操作审计日志（本地可检索）。

**验收标准**
- 三平台安装包可用。
- 关键操作（添加账号、切换默认、导出导入）均可在 GUI 完成。

---

## Milestone 4: 远程项目与虚拟环境执行层
**目标**：让 AI CLI 在远端工程环境稳定运行，本地只做控制与观察。

**范围**
- 远程 Agent Daemon（Linux 优先）。
- 远程项目工作区绑定（多项目隔离）。
- 本地 CLI 直连远程项目：在本地执行 `aih codex|claude|gemini`，实际在远程项目目录中运行并改代码。
- 虚拟环境执行层（容器/沙箱），支持 codex/claude/gemini 在隔离环境运行。
- 传输与控制通道（WebSocket/gRPC），包含鉴权与会话续连。
- 远程文件同步与变更回传（最小可用版先支持 Git 仓库目录与 patch 回传）。

**验收标准**
- 本地 `aih` 可连接远端并启动指定项目会话。
- 本地 codex 可直接开发远程项目（创建/修改文件、运行命令、查看结果）。
- 远端断连可重连恢复，不丢控制上下文。
- 工具运行环境可复现（镜像/环境模板可重建）。

---

## Milestone 5: 手机端指挥中心
**目标**：手机端进行“控制与追踪”，不在手机本地跑重型开发环境。

**产品定位**
- 手机端不是“本地运行器”，而是“远程控制终端”。
- 角色更接近 Termius / HappyCode 这类工具：连接远端、发指令、看结果、做运维动作。
- 成败关键是移动端体验，而不是堆叠本地执行能力。

**范围**
- iOS/Android 客户端（建议 React Native）。
- 连接远端 Daemon，查看会话状态、触发任务、查看结果摘要。
- 轻量交互模式：消息流 + 关键日志 + 操作按钮。
- 推送通知：任务完成、失败告警、额度耗尽。
- 不在手机端安装或运行 codex/claude/gemini 主执行环境（统一由远端执行层承担）。

**体验优先级（按先后顺序）**
1. 连接稳定性与重连速度。
2. 小屏信息层级（关键状态优先，日志可折叠）。
3. 单手操作路径（常用动作 1-2 步完成）。
4. 低延迟反馈（发起后秒级看到任务状态变化）。
5. 异常可恢复（断网、超时、权限失败都给明确下一步）。

**验收标准**
- 手机端可发起一次完整任务并看到结果回执。
- 常见运维动作（重试、停止、切账号）可在手机端完成。
- 对比桌面端，手机端核心路径操作步数不超过 +1 步。

## 你原始目标的对齐结论
已覆盖且无遗漏：
1. 加密导出导入并跨系统恢复（仅账号配置资产）。
2. Rust 优先 GUI + 三平台支持。
3. 远程项目支持。
4. 手机端支持。
5. 虚拟环境运行 codex/claude/gemini。

当前唯一需要你拍板的策略点：
- 虚拟环境执行层首版是“容器优先（Docker/Podman）”还是“轻沙箱优先（chroot/nsjail）”。

## 里程碑拆解（执行版）
为避免“方向正确但落地缓慢”，按可交付拆成以下小里程碑：

### M1.1（CLI 会话基线）
- 完成单一全局会话源稳定化。
- 完成默认切换最小化（不劫持全局目录）。
- 输出：稳定、无副作用的 CLI 基线版本。

### M1.2（诊断与可观测）
- 增加 `aih doctor` 和关键状态检查项。
- 增加关键操作审计日志（CLI 层）。
- 输出：故障自查与问题定位能力。

### M2.1（导出导入非交互化）
- 导出导入支持脚本参数化（非交互）。
- 冲突策略标准化（skip/overwrite/report）。
- 输出：可在自动化流程中使用的迁移能力。

### M2.2（导入后可用性验证）
- 导入后自动校验账号可用性（轻量启动检查）。
- 输出：跨机恢复后“可直接工作”的确定性。

### M3.1（桌面端 MVP）
- Rust + Tauri 工程骨架。
- 账号看板 + 启动会话 + 默认切换。
- 输出：桌面端首个可用版本（单机）。

### M3.2（桌面端增强）
- 图形化导出导入与日志查看。
- 三平台安装包流水线。
- 输出：可分发的桌面版本。

### M4.1（远程连接层）
- 远程 Daemon + 鉴权 + 连接管理。
- 输出：本地可稳定连接远端节点。

### M4.2（远程项目开发能力）
- 本地 codex/claude/gemini 直连远程项目目录。
- 支持文件改动、命令执行、结果回传。
- 输出：本地开发远程项目的完整闭环。

### M4.3（虚拟环境执行层）
- 远程执行环境模板化（容器或轻沙箱）。
- 输出：可复现、可隔离的远程运行环境。

### M5.1（手机端 MVP）
- 登录与节点连接。
- 会话状态查看、任务发起、结果查看。
- 输出：可远程指挥的手机端基础版。

### M5.2（手机端体验优化）
- 断线重连、推送通知、单手高频操作优化。
- 输出：接近 Termius/HappyCode 水平的移动端体验。
